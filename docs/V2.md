# V2 — Architecture & Modernization

## What Changed

### 1. Dead Code Removal

All original root-level files were superseded by the `poker/` package and have been deleted:

| File | Lines | Reason |
|---|---|---|
| `misc.py` | 479 | Old support library (cards, ranges, evaluation, equity, table rendering) |
| `poker_cli.py` | 311 | Monolithic CLI game, superseded by `poker/` package + GUI |
| `poker_gui.py` | 523 | Monolithic PySimpleGUI game, superseded by `poker/gui.py` |
| `player.py` | 14 | Old Player class, superseded by `poker/player.py` (dataclass) |
| `main.py` | 6 | Stub, replaced with actual entry point |
| `hv.json` | 48 MB | Pre-computed hand value table, superseded by algorithmic + pokerkit evaluation |
| `requirements.txt` | 288 | Frozen environment cruft, superseded by `pyproject.toml` + `uv.lock` |

**Total removed: ~1,600 lines of Python + 48 MB JSON.**

### 2. Python 3.13 Native

- Removed `from __future__ import annotations` from all `poker/` modules (PEP 604 unions, deferred evaluation, and all 3.13 features are now native).
- Added `strict=True` to all `zip()` calls in `game.py` for safety.
- `pyproject.toml` targets `>=3.13`.

### 3. Architecture

```
main.py                          ← single entry point
 └─ poker.gui.main()
     ├─ PokerGUI (tkinter)       ← implements GameUI protocol
     ├─ TexasHoldem (game.py)    ← UI-agnostic engine
     │   └─ evaluation.py        ← backed by pokerkit StandardHighHand
     ├─ equity.py                ← Monte Carlo equity simulation
     ├─ ranges.py                ← pre-flop hand rankings
     ├─ cards.py                 ← card primitives, 52-card deck
     └─ player.py                ← Player dataclass
```

### 4. Hand Evaluation — Pokerkit Integration

`evaluation.py` was rewritten to use `pokerkit.StandardHighHand` instead of a custom algorithmic evaluator.

| Before | After |
|---|---|
| Custom `Counter`-based 5-card evaluation | `StandardHighHand` with `StandardLookup` |
| Manual `C(n,5)` max over combinations | `max(combinations(cards, 5), key=StandardHighHand)` |
| Hand category as first tuple element | `value.entry.label` → `Label.STRAIGHT_FLUSH` etc. |
| `HandValue = tuple[int, ...]` | `HandValue = StandardHighHand` (directly comparable) |

Benefits:
- Battle-tested evaluation from a dedicated poker library (forked from `uoftcprg/pokerkit`)
- Built-in hand descriptions (e.g. `"Full house"`, `"Straight flush"`)
- Unique ranking indices via `entry.index`
- Eliminates 48MB hv.json dependency

### 5. GUI — Zero-Dependency Tkinter

The old PySimpleGUI + matplotlib + PIL GUI was replaced with a pure tkinter implementation (~250 lines).

**Layout:**
- **Player area** — names, chip counts, and card displays for each player
- **Board area** — 5 community card slots with suit-colored text (♠♥♦♣)
- **Pot display** — live pot size
- **Hand evaluation** — gold label showing pokerkit's hand name at showdown (e.g. "Full house")
- **Action buttons** — Check/Call, Raise (with slider), Fold
- **Game log** — scrollable text log showing all events

**Integration pattern:** The `GameUI` protocol's `request_bet()` uses `tk.wait_variable()` to block without freezing the event loop, enabling the synchronous game engine to drive an event-driven GUI.

### 6. Entry Point — Single File

```
python main.py
```

Supports optional arguments: `-n/--players`, `-b/--buy-in`, `--big-blind`, `--seed`, `--names`.
