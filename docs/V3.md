# V3 — Agent Framework (smolagents)

## What Changed

### 1. New Package: `agent/`

A new top-level `agent/` package was added, providing a framework for AI-powered poker agents using HuggingFace's [smolagents](https://huggingface.co/docs/smolagents) library.

```
agent/
├── __init__.py           # Public API exports
├── game_state.py         # GameState, PlayerInfo, Action dataclasses
├── tools.py              # 6 @tool-decorated functions for agents
├── harness.py            # AgentHarness + GameDirector
├── prompts.py            # System prompt templates for play styles
└── replay.py             # ReplayLog for spectator replay
```

### 2. Architecture

```
TexasHoldem (poker/game.py)
  └─ ui: GameDirector (agent/harness.py)   ← implements GameUI Protocol
      ├─ AgentHarness (per player)
      │   └─ CodeAgent (smolagents)
      │       └─ tools: hand rank, equity, pot odds, position, etc.
      └─ ReplayLog
```

The `GameDirector` implements the existing `GameUI` Protocol, making it a drop-in replacement for the Flet GUI. It:

- **Parses engine messages** to track hand number, street, community cards, blind postings, hole cards, and chip counts.
- **Routes `request_bet()`** to the correct `AgentHarness` for the acting player.
- **Builds filtered `GameState`** per agent — each agent only sees its own hole cards.
- **Records every action** to a `ReplayLog` for later playback.

### 3. Game State Model (`game_state.py`)

| Dataclass | Fields |
|---|---|
| `GameState` | `hand_number`, `street`, `pot`, `to_call`, `max_bet`, `hero_name`, `hero_cards`, `hero_chips`, `hero_position`, `num_players`, `community_cards`, `players: list[PlayerInfo]`, `actions: list[Action]` |
| `PlayerInfo` | `name`, `chips`, `folded`, `bet` |
| `Action` | `player`, `action_type` (fold/check/call/raise/blind), `amount`, `street` |

### 4. Agent Tools (`tools.py`)

Six smolagents `@tool`-decorated functions that agents call during decision-making:

| Tool | Returns | Description |
|---|---|---|
| `get_hand_rank()` | `str` | Pre-flop class (e.g. "AKo") or post-flop category ("two pair") |
| `calculate_equity(n)` | `float` | Monte Carlo win probability (0.0–1.0) via `poker.equity.simulate_equity` |
| `get_pot_odds()` | `float` | `call / (pot + call)` ratio |
| `get_position()` | `str` | "small blind", "big blind", "early", "middle", "late", "dealer (button)" |
| `get_preflop_class()` | `str` | 169-class label like "AKs", "TT", "76o" |
| `get_game_context()` | `str` | Full formatted snapshot of the game state |

Tools read from a module-level `_current_state` variable that the harness sets before each agent run — no threading conflicts.

### 5. Agent Harness (`harness.py`)

**`AgentHarness`** — wraps a smolagents `CodeAgent` per player:

```python
harness = AgentHarness(
    name="BotA",
    model=InferenceClientModel(model_id="Qwen/Qwen2.5-Coder-7B-Instruct"),
    style="TAG",
    tools_list=[...],
    max_steps=8,
)
harness.decide(state)  # → bet amount (int)
```

- On `decide()`, builds a task prompt, sets tool state, runs `agent.run(reset=True)`, and parses the output via `_parse_bet()`.
- `_parse_bet()` handles free-text LLM output: "fold", "call", "raise to 40", "all in", raw numbers, etc., with clamping and fallback to fold.
- `from_model_id()` factory for quick setup.
- Try/except wraps `agent.run()` — any failure results in fold (0).

**`GameDirector`** — implements `GameUI` to orchestrate multiple agents:

```python
director = GameDirector(harnesses, player_names, buy_in=200)
game = TexasHoldem(players, ui=director, big_blind=2)
winner = game.play()
print(director.replay.summary())
```

- Tracks internal state from engine messages (hole cards, community cards, blinds, chip updates).
- Builds a per-agent `GameState` snapshot on each `request_bet()` call.
- Validates bets, tracks folds, classifies actions, records replay steps.
- Raises `RuntimeError` if a player has no configured agent.

**`create_harnesses()`** — factory for bulk agent creation with optional per-name style mapping.

### 6. Play Styles (`prompts.py`)

Five built-in styles with distinct strategy instructions:

| Style | Personality |
|---|---|
| **TAG** | Tight-Aggressive — premium hands only, aggressive with strength |
| **LAG** | Loose-Aggressive — wide range, constant pressure, frequent bluffs |
| **CallingStation** | Passive — calls everything, never raises, never bluffs |
| **Random** | Unpredictable — erratic decisions |
| **Nit** | Extreme tight — only AA, KK, QQ, AK |

The `BASE_SYSTEM_PROMPT` template includes tool descriptions and strict decision rules (0 = fold/check, call_amount = call, > call_amount = raise).

### 7. Replay System (`replay.py`)

```python
log = ReplayLog()
log.record(ReplayStep(hand_number, street, player, action_type, amount, state))
log.step_forward()   # → ReplayStep | None
log.step_back()      # → ReplayStep | None
log.summary()        # → list[str]
```

Records every action with a full `GameState` snapshot for later step-through replay.
